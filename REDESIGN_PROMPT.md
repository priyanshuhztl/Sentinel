# Token Watcher UI — Vercel Design System Redesign Prompt

> **Purpose:** Complete visual redesign of the Token Watcher dashboard using the Vercel/Geist design system. All existing features and data flows remain unchanged — this is a pure visual/styling overhaul.
>
> **Source of truth for design tokens:** `DESIGN.md` (generated by `npx getdesign@latest add vercel`).

---

## 0. What This App Does (Don't Change Any of This)

Token Watcher UI is a real-time dashboard monitoring Claude Code API token usage. It:

- Watches `~/.claude/projects/*.jsonl` files and parses Claude Code session logs
- Streams live data to the browser via SSE at `/api/stream`
- Shows two tabs: **Dashboard** and **Prompt Health**
- **Dashboard tab** — 3-column grid:
  - `CurrentChat`: active session token breakdown + costs
  - `UsageLimitsCard`: Anthropic API usage % progress bars
  - `AllChats`: recent session list
  - `HourlyBreakdown`: last 6-hour cost/output bars
  - `RecentRequests`: last 20 API calls table
- **Prompt Health tab** — cache efficiency metrics + expandable session/prompt quality table
- `Header`: sticky top bar with live status, summary stats, plan dropdown
- `PlanSettings`: dropdown that sets subscription plan (saved to localStorage)

**Do not change** any logic in `lib/`, `hooks/`, `app/api/`, or `instrumentation.ts`. Only change styling in `app/globals.css`, `app/layout.tsx`, `app/page.tsx`, and all files in `components/`.

## 3. Typography Rules

| Token | Size | Weight | Family | Use in this app |
|---|---|---|---|---|
| `display-md` | 24px / 600 / -0.96px ls | Geist Sans | Section titles (rare, hero stats only) |
| `display-sm` | 20px / 600 / -0.6px ls | Geist Sans | Big metric numbers (cost, totals) |
| `body-sm-strong` | 14px / 500 | Geist Sans | Card section labels, column headers, button text |
| `body-sm` | 14px / 400 | Geist Sans | Default body, table cell values, descriptions |
| `caption` | 12px / 400 | Geist Sans | Timestamps, secondary metadata |
| `caption-mono` | 12px / 400 | **Geist Mono** | Table column headers, token counts, request IDs, model names, flag codes |
| `code` | 13px / 400 | **Geist Mono** | Session IDs, file paths, prompt text previews |

**Critical rules:**
- **Never use font-weight 700+** anywhere in the UI
- Use Geist Mono for: all table column headers, all token counts, model names, session IDs, prompt text, score numbers
- Use Geist Sans for: all prose text, labels, tabs, buttons, navigation
- Large metric values (e.g., `$0.0824`, `42,381 tokens`) use `display-sm` + Geist Mono + `tabular-nums`

## 8. What NOT to Do

- **No dark background** — the redesign is light mode (canvas white / canvas-soft #fafafa)
- **No weight 700+** — max weight is 600 (display headings only), body is 400–500
- **No colored card backgrounds** — all cards are white (#ffffff) with shadow, never colored bg
- **No single `box-shadow: 0 4px 6px rgba(0,0,0,0.1)` shortcuts** — always use the stacked shadow vars
- **No pill radius (100px) on dashboard buttons** — pill radius is marketing-only; use `--radius-sm` (6px)
- **No inline color hex values** — use the CSS variables defined in globals.css
- **No amber/red/green for general accents** — only use them for their semantic meaning (cost=amber, success=green, error=red)
- **No Tailwind color classes like `text-yellow-400`** — use inline CSS `color: var(--color-amber-700)` or custom Tailwind token
- **Never bold the muted/secondary text** — only `--color-ink` text can be `font-weight: 500+`

## 9. Gradient Usage (Strictly Limited)

The Vercel mesh gradient is the brand's **only decorative element**. Use it exactly once in the UI:

**Where:** A 3px horizontal stripe at the very top of the `<Header>` (above the nav content), spanning full width.

```css
.header-gradient-stripe {
  height: 3px;
  background: var(--gradient-mesh);  /* #007cf0 → #00dfd8 → #7928ca → #ff0080 */
  width: 100%;
}
```

Nowhere else. No gradient fills on cards, badges, buttons, or backgrounds.
